Выбор инструмента зависит от требований проекта и опыта команды. Вот честное сравнение с популярными Go scaffolding и framework решениями.

## Когда выбрать Go Project Starter

### Идеально подходит для:

- Создания бизнес-критичных сервисов с REST, gRPC и async workers с нуля
- Команд, которым нужна консистентность между множеством микросервисов
- Проектов, требующих частой регенерации при эволюции API
- Сервисов со сложными инфраструктурными потребностями (multi-transport, несколько БД)
- Организаций, стандартизирующих clean architecture паттерны

### Рассмотрите альтернативы если:

| Если вам нужно | Рассмотрите |
|----------------|-------------|
| GUI-based low-code инструмент | **Sponge** |
| Лёгкий фреймворк для изучения микросервисов | **go-kit** |
| Только gRPC-to-REST gateway | **grpc-gateway** напрямую |
| DSL вместо YAML конфигурации | **Goa** |
| Ультра-высоконагруженные сервисы | **go-zero** |

## Детальное сравнение

### Goa

**Goa** — design-first фреймворк с собственным DSL.

| Плюсы | Минусы |
|-------|--------|
| Мощный DSL для описания API | Нужно изучать новый DSL |
| Нативная генерация OpenAPI | Нет сохранения пользовательского кода |
| Хорошая документация | Минимальная инфраструктура |

**Когда выбрать Goa:**
- Команда готова инвестировать в изучение DSL
- Нужен design-first подход к API
- Инфраструктура уже готова

### go-zero

**go-zero** — cloud-native микросервисный фреймворк от Alibaba.

| Плюсы | Минусы |
|-------|--------|
| Высокая производительность | Только одноразовая генерация |
| Встроенный service mesh | Framework lock-in |
| etcd/consul интеграция | Более сложный порог входа |

**Когда выбрать go-zero:**
- Строите high-traffic сервисы
- Нужен service mesh из коробки
- Работаете с etcd/consul

### Sponge

**Sponge** — low-code инструмент с GUI.

| Плюсы | Минусы |
|-------|--------|
| GUI интерфейс | Ограниченное сохранение кода |
| Встроенный AI copilot | Меньше гибкости |
| Быстрый старт | Ориентирован на CRUD |

**Когда выбрать Sponge:**
- Нужен визуальный интерфейс
- Строите простые CRUD сервисы
- Хотите AI-помощника

### grpc-gateway

**grpc-gateway** — прокси для трансляции REST в gRPC.

| Плюсы | Минусы |
|-------|--------|
| Простой и фокусированный | Только gateway функционал |
| Стабильный и зрелый | Нет инфраструктуры |
| Широко используется | Перезапись при регенерации |

**Когда выбрать grpc-gateway:**
- Уже есть gRPC сервис, нужен REST
- Нужен только gateway
- Инфраструктура уже готова

### go-kit

**go-kit** — toolkit для построения микросервисов.

| Плюсы | Минусы |
|-------|--------|
| Максимальная гибкость | Высокий порог входа |
| Отличные паттерны | Много boilerplate кода |
| Transport-agnostic | Нужно много писать вручную |

**Когда выбрать go-kit:**
- Нужен полный контроль над архитектурой
- Команда опытна в микросервисах
- Готовы писать больше кода

## Уникальные преимущества Go Project Starter

### 1. Сохранение пользовательского кода

**Единственный генератор**, который позволяет перегенерировать проект без потери ваших изменений:

```go
// ==========================================
// GENERATED CODE - DO NOT EDIT ABOVE THIS LINE
// ==========================================

func (h *Handler) CreateUser(...) {
    // Ваш код здесь - переживёт регенерацию!
}
```

### 2. Application-based масштабирование

Одна кодовая база → разные профили деплоя:

```yaml
applications:
  - name: api
    transport: [rest, grpc]

  - name: workers
    workers: [telegram_bot]
    kafka: [events]
```

### 3. Полная инфраструктура из коробки

- Docker multi-stage builds
- GitHub Actions CI/CD
- Traefik reverse proxy
- Prometheus метрики
- Grafana dashboards
- Structured logging
- Graceful shutdown

### 4. OpenAPI-first с escape hatches

- Генерация из OpenAPI через ogen
- Кастомные endpoints через template генератор
- Смешивание разных генераторов

### 5. Production-ready с первого дня

- ~50 файлов, ~8000 строк кода
- Компилируется с первого раза
- Нулевая конфигурация для старта

## Миграция на Go Project Starter

### С Goa

1. Экспортируйте OpenAPI из Goa DSL
2. Создайте config.yaml с path к OpenAPI
3. Запустите генератор
4. Перенесите бизнес-логику под disclaimer-маркеры

### С go-zero

1. Экспортируйте API спецификации
2. Создайте config.yaml
3. Запустите генератор
4. Адаптируйте код под новую структуру

### С grpc-gateway

1. Используйте существующие .proto файлы
2. Добавьте OpenAPI спецификации для REST
3. Создайте config.yaml
4. Генерируйте комбинированный сервис
